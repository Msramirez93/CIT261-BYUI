JavaScript Events

Event Handlers

JavaScript in the browser uses an event-driven programming model.

Everything starts by following an event.

Sometimes, events handlers are referred to as event listener, however some programmers see them as two different handlers:

  1. Event Handler
    2. A JavaScript function that runs when an event fires.
  2. Event Listener
    3. A element that waits and "listen" for a given event to fire.

Examples of different event handlers are:

  1. Inline Event Handlers
    2. <button onclick="document.getElementById('demo'). innerHTML=Date()">The time is?</button>
  2. Dom On-Event Handlers
    3. <onload="myFunction()">
  3. Event Listeners
    4. window.addEventListener('load', () =>{ //window loaded})

Dom On-Event Handlers

Syntax:

Using an HTML attribute named on{eventtype} on an element, for example:

<button onclick="return handleClick(event);"></button>

Or by setting the corresponding property from JavaScript, for example:

document.getElementById("mybutton").onclick = function(event) { ... }

Event Listeners

The latest addition to JavaScript event handlers are event listeners. An event listener watches for an event on an element. Instead of assigning the event directly to a property on the element, we will use the addEventListener() method to listen for the event.

Syntax:

element.addEventListener(event, function, useCapture)
